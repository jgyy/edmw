One of the big issue is the work flow.

Customer : how much total?
Stall : $2.70 (verbally)
Customer and or stall need to key in that amount.
Two things need to happen. Keyed amount to be correct. Transaction needs to go thru.
If customer key. Its a workload to him/her.
If stall key. Its additional load on top of verbal.
Then there's the wait for transaction to go thru n verification.
Recept? Paper? Ecopy? All these takes time.

On cash. $2.70? K, customer pass $2 note, $1 coin. Stall returns 30 cents. Done deal. Move on and next. Note the transaction speed, n fuss free.

* granted, no transaction record on either stall/customer. But is there a pressing need to record this? Like for example of a user going to the toilet how many times a day, to pee or poo, pee how much, poo how heavy thing? As in for everyone?
