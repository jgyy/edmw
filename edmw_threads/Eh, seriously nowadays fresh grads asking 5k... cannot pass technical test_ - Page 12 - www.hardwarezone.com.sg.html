Well the first post certainly didn’t talk about having different URLs. If that is the case, it is also very simple.

Simply replace the curl command with the following variant

Code: [ -f /tmp/download.url ] && curl -s -L -O `cat /temp/download.url`
If it is a web interface that determine which url to use, just write that url into /tmp/download.url.

The technique above will work with a lot of ways to trigger and shell script with the abundance of unix tools has so much power to adapt and integrate. You can even read from a JSON file if you like with tools like *jq*
fun challenge time:

1) the website requires you to login

2) the server returns you a cookie after logging in.

3) the web client generates a CSRF token based on the cookie using a javascript function embedded with the .html webpage

4) you must be authenticated before you can get the server to generate the DL link. the server validates your identity using both the cookie and the CSRF token

5) there are no public APIs available. so no providing API keys or JWTs to gain access. there is only the web portal available so you must handle authentication with cookie + CSRF token

how will you handle this?
